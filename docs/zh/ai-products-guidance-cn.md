# AI 编程产品指导文件说明

本文档说明了为主要 AI 编程产品生成的指导文件。

## 背景

随着 AI 编程工具的普及，不同的 AI 产品需要不同格式的配置文件来理解项目的结构、规范和最佳实践。本项目为以下主要 AI 编程产品提供了专门的指导文件：

## 已支持的 AI 产品

### 1. GitHub Copilot
- **文件位置**: `.github/copilot-instructions.md`
- **状态**: ✓ 已存在
- **说明**: GitHub 官方 AI 编程助手，已有完整的中文指导文件

### 2. Cursor
- **文件位置**: `.cursorrules`
- **状态**: ✓ 新增
- **说明**: AI-first 代码编辑器，基于 VSCode
- **特点**: 自动读取项目根目录的 `.cursorrules` 文件

### 3. Windsurf  
- **文件位置**: `.windsurfrules`
- **状态**: ✓ 新增
- **说明**: Codeium 推出的 AI 编程编辑器
- **特点**: 强大的代码理解和生成能力

### 4. Cline (formerly Claude Dev)
- **文件位置**: `.github/cline-instructions.md`
- **状态**: ✓ 新增
- **说明**: VSCode 扩展，支持复杂多步骤任务
- **安装**: VSCode 扩展市场搜索 "Cline"

### 5. Aider
- **文件位置**: `.aider.conf.yml`
- **状态**: ✓ 新增
- **说明**: 命令行 AI 编程工具
- **安装**: `pip install aider-chat`
- **使用**: 在项目根目录运行 `aider`

## 使用方法

### 对于开发者

1. **选择 AI 工具**: 根据你的偏好选择一个或多个 AI 编程工具
2. **安装工具**: 按照各工具的官方文档安装
3. **自动生效**: 工具会自动读取相应的配置文件
4. **开始编码**: AI 助手会根据项目规范提供建议

### 对于 AI 工具

配置文件为 AI 提供了以下信息：
- 项目结构和模块组织
- 代码风格和命名规范
- 构建和测试命令
- 常见问题和解决方案

这使得 AI 能够：
- 生成符合项目规范的代码
- 提供准确的构建和测试命令
- 理解项目的架构和依赖关系
- 给出符合上下文的建议

## 维护指南

### 何时更新

当以下情况发生时，应更新相应的指导文件：

1. **架构变更**: 添加/删除模块、重大 API 变更
2. **规范变更**: 代码风格、命名约定更新
3. **工具升级**: Maven 版本、构建脚本变化
4. **流程优化**: 测试策略、CI/CD 流程调整

### 如何更新

1. 确定需要更新的内容
2. 同时更新所有相关文件：
   - `.github/copilot-instructions.md`
   - `.cursorrules`
   - `.windsurfrules`
   - `.aider.conf.yml`
   - `.github/cline-instructions.md`
3. 更新版本号（当前 v2025-10-14）
4. 提交 PR 并说明变更原因

### 文件同步

**重要**: 所有 AI 产品的指导文件应保持内容一致，只是格式可能因工具而异。

- Markdown 文件（Copilot, Cursor, Windsurf, Cline）使用相同的结构化文本
- YAML 文件（Aider）使用结构化配置，但包含相同的信息

## 扩展新的 AI 产品

如需为新的 AI 编程产品添加支持：

1. **研究工具**: 了解该工具如何读取项目配置
2. **确定格式**: 确定配置文件的格式和位置
3. **创建文件**: 基于现有模板创建新的配置文件
4. **测试验证**: 确保工具能正确读取配置
5. **更新文档**: 在 `docs/ai-products-guidance.md` 中添加说明
